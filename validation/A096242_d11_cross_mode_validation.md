# A096242 D11 Cross-Mode Validation

**Date:** June 7, 2026  
**Scope:** production-program rerun of the first new level, `D11`, in two deliberately different execution modes.

## Purpose

Paths such as `levels/D11.bin` refer to saved binary level files generated
during the local computational campaign.  These binary files are not included
in the minimal GitHub package; their counts, byte sizes, and SHA-256 hashes
are documented in `levels_manifest.tsv`.

This check was added after manuscript review identified that the saved-level
certifier verifies validity of saved levels but does not independently
re-enumerate missing candidates.  The goal here is narrower: verify that the
production implementation gives a byte-identical `D11.bin` file under two
different production modes.

## Commands

Single-thread in-memory rerun:

```sh
/usr/bin/time -v ./build/a096242 \
  --max-n 11 \
  --threads 1 \
  --mode memory \
  --validate \
  --bfile data/b096242.txt \
  --tmp-dir tmp/d11_crosscheck_20260607_094858/tmp_mem \
  --save-levels tmp/d11_crosscheck_20260607_094858/levels_mem
```

Forced segmented rerun:

```sh
/usr/bin/time -v ./build/a096242 \
  --max-n 11 \
  --threads 16 \
  --mode segmented \
  --chunk-parents 10000 \
  --validate \
  --bfile data/b096242.txt \
  --tmp-dir tmp/d11_crosscheck_20260607_094858/tmp_seg \
  --save-levels tmp/d11_crosscheck_20260607_094858/levels_seg
```

## Results

Both runs reproduced all known terms through `n=10` and produced:

```text

```

The single-thread in-memory rerun reported:


The 16-thread forced segmented rerun reported:

## Byte-Identity Check

The saved production file, the memory-mode rerun file, and the segmented rerun
file had the same SHA-256 hash:

```text
38ffc99bf4836f551e856eced2079d2c841dd1d8e721e600bf4e119af0dd3a59  levels/D11.bin
38ffc99bf4836f551e856eced2079d2c841dd1d8e721e600bf4e119af0dd3a59  tmp/d11_crosscheck_20260607_094858/levels_mem/D11.bin
38ffc99bf4836f551e856eced2079d2c841dd1d8e721e600bf4e119af0dd3a59  tmp/d11_crosscheck_20260607_094858/levels_seg/D11.bin
```

The regenerated `D11.bin` files are not included in the minimal GitHub
package.  They are generated artifacts and can be regenerated from the public
source and compared against `levels_manifest.tsv`.

## Interpretation

This is a production cross-mode validation of `D11`: the first new level was
regenerated by the same production source in both memory and segmented modes,
with byte-identical output.  It is not an independent re-enumeration by a
separate implementation, and it does not extend to `D12`, `D13`, or `D14`.
